# Glossary of Terms

| Term                             | Definition                                                                                        |
|----------------------------------|---------------------------------------------------------------------------------------------------|
| RPE (Rate of Perceived Exertion) | A subjective 1–10 scale measuring how difficult a set feels. 1 = very light; 10 = maximal effort. |
| 1RM (One‑Rep Max)                | The maximum amount of weight a user can lift for one repetition of an exercise.                   |
| Set                              | A group of consecutive repetitions of an exercise.                                                |
| Rep (Repetition)                 | One complete movement of an exercise.                                                             |
| Superset                         | Two or more exercises performed back‑to‑back with minimal rest.                                   |
| Circuit                          | A sequence of exercises performed in rotation with limited rest.                                  |
| Program                          | A structured multi‑week training plan consisting of workouts, days, and exercises.                |
| Workout                          | A single training session containing one or more exercises.                                       |
| Exercise Summary                 | A short description of an exercise including cues, equipment, and instructions.                   |
| PR (Personal Record)             | The highest weight, reps, or performance achieved by a user for a given exercise.                 |
| Previous Performance             | View previous logged exercise metrics to inform progression over time                             |
|Accessibility|Design practices ensuring the app is usable by people with disabilities, including screen reader support, keyboard navigation, ARIA roles, and reduced‑motion compliance.|
|Action Button|A button that triggers a primary or secondary action on a screen (e.g., “Login,” “Add Entry,” “Start Workout”).|
|ARIA (Accessible Rich Internet Applications)|A set of attributes that improve accessibility for users relying on assistive technologies.|
|Card|A container component used to display grouped information such as program summaries, insights, or notifications.|
|Cardio Exercise|An exercise that is intended for Cardio Vascular Health (aka, running, rowing, etc.)|
|Confirmation Modal|A focus‑trapped dialog requiring explicit user confirmation before performing a destructive or irreversible action (e.g., deleting a weight entry).|
|Consistency Trend|A graph or insight showing how frequently the user logs workouts over time.|
|Dashboard|The authenticated landing page shows user identity, program summary, weight insights, recent activity, and trend graphs.|
|Day Detail|The expanded view within a program week showing all exercises scheduled for a specific day.|
|Destructive Action|An action that cannot be undone (e.g., deleting a notification or weight entry).|
|Empty State|A UI pattern displayed when no data exists for a given screen or component (e.g., “No notifications,” “No weight entries yet”).|
|Error Banner|A non‑blocking message displayed at the top of a screen or section when an operation fails.|
|Exercise Card|A card that represents a single exercise within a workout or program day.|
|Focus Trap|A behavior that restricts keyboard focus to a modal, menu, or expanded container until it is closed.|
|Form Validation|Rules that ensure user input is complete and correct before submission.|
|Graph Skeleton|A placeholder block shown while trend graphs are loading.|
|Group (Aria Role)|A semantic grouping of related content, often used for expanded notifications or day detail sections.|
|Hamburger Menu|The authenticated navigation menu, accessed from the header icon.|
|Header|The persistent top bar, containing the NeuByte logo and, when authenticated, the Notification Icon and Hamburger Menu.|
|Inline Error|A validation message displayed directly beneath an input field.|
|Insight|A small, data‑driven summary such as weight change, consistency streaks, or program progress.|
|List Row|A row representing a single item in a list (e.g., notification, weight entry, recent activity).|
|Loading Indicator|A spinner or animation shown during server‑side operations such as saving or retrieving data.|
|Mark All as Read|A notification action that marks all unread notifications as read.|
|Metadata (Program Metadata)|Descriptive information about an exercise program, including name, duration, difficulty, and description.|
|Notification|A system‑generated message informing the user of activity, reminders, or updates.|
|Notification Icon|The bell icon in the authenticated header, showing unread count and opening the Notification List.|
|Program Detail|The screen, showing the full structure of an exercise program, including weeks, days, and exercises.|
|Program Summary|A card on the Dashboard showing the user’s currently selected program.|
|Read/Unread State|The visual and semantic state of a notification indicating whether it has been viewed.|
|Report|A Power BI–based analytical view showing trends and insights derived from user data.|
|Retry Action|A button that reattempts a failed operation (e.g., loading notifications, retrieving program data).|
|Screen‑Level Error|A full‑screen error state is shown when a screen cannot load required data.|
|Skeleton Loader|A placeholder UI element shown while content is loading.|
|Strength Exercise|Exercise that requires sets, reps, and weight inputs.|
|Trend Graph|A visual representation of changes over time(e.g., weight trend, consistency trend)|
|Timestamp|The date and time associated with a notification, weight entry or activity.|
|Unauthenticated Screen|A screen accessible without login (Login, Create Account, Reset Password).|
|Unread Indicator|A visual marker (dot or bold text) showing that a notification has not been read.|
|Week Selector|A list of week buttons used to navigate program weeks.|
|Weight Entry|A single recorded body weight value with timestamp.|
|Workout Overview|The summary at the top of the Log Workout screen showing today’s workout context.|
